## 2. Repository Architecture Discovered

**Frontend:** Next.js 14 App Router, TypeScript, TailwindCSS, shadcn/ui, Recharts. Dark Observatory design system (`#0A0A0F` base, indigo-violet gradient).

**Backend:** Next.js API Routes (~60 routes). Middleware enforces auth + CSRF centrally. Rate limiting via Upstash Redis (fail-closed).

**Database:** Supabase (PostgreSQL). 30 migrations. RLS on all tables. Service-role client for Worker callbacks and cron jobs.

**Worker:** Cloudflare Worker (`src/workers/analyzer/index.ts`). Fetch-only — no Playwright in the Worker; analysis is done server-side via HTML parsing + Claude Vision. The `screenshot.ts`/`lighthouse.ts` sub-files referenced in CLAUDE.md do not exist.

**Queue:** Upstash Redis. 13 job types, 9-state lifecycle, distributed lease, per-origin staggering, exponential backoff, DLQ. Cron scheduler at `/api/cron/queue-scheduler` (promotes scheduled→ready). Consumer at `/api/cron/queue-consumer` (claims and executes). Monitor dispatcher at `/api/cron/monitors` (creates analysis records + enqueues jobs with per-origin delay).

**Public SDKs:** Connected Sites browser script generated by `src/lib/site-connect/script-source.ts`. No standalone error monitoring SDK exists.

**Plans:** Free (3 credits/mo), Pro, Agency, Compliance. 27+ feature flags centralized in `src/lib/billing/limits.ts`. Stripe webhooks update `subscriptions` table. Credits consumed atomically via `use_credit()` DB function.
